## 03 · 权限解释

> Dev Console "隐私惯例" 区每个权限都需要单独填写"用途说明"。  
> 以下文案为已通过审核的精简表述，**逐字粘贴**到对应权限框即可。

---

### `tabs`

```
读取当前打开标签页的 URL、标题、favicon，用于在侧边栏展示标签卡片，并实现按域名分组、去重、按最近访问排序等核心功能。不会修改任何标签内容，不会注入脚本到网页中。
```

### `tabGroups`

```
执行"一键整理"时，将同一域名的多个标签合并到 Chrome 原生标签组（用户可见的彩色折叠条），方便用户在原生标签栏中也能直观看到分组结果。
```

### `sessions`

```
实现"撤销关闭"功能。当扩展自动或用户主动关闭重复标签后，弹出 Toast 通知；点击 Undo 时通过 chrome.sessions.restore 恢复刚关掉的标签。
```

### `storage`

```
本地保存用户偏好：白名单（永不自动关闭的 URL 列表）、主题（浅色/深色）、去重模式（Strict/Loose/Domain）、最近访问时间戳缓存。仅使用 chrome.storage.local，所有数据不会离开设备。
```

### `sidePanel`

```
将 CosmoTab 的主界面 (Dashboard) 渲染在 Chrome 侧边栏，使用户无需打开新标签页即可快速浏览和管理所有打开的标签。
```

### `favicon`

```
通过 chrome://favicon/ 拉取每个标签所属网站的图标，用于在标签卡片左侧显示 16×16 站点 logo，提升识别效率。
```

### 关于 host_permissions

CosmoTab 当前**不申请任何 host_permissions**（即不申请 `<all_urls>` / 具体域名）。  
如未来需要在网页中注入脚本（暂无此计划），需在隐私惯例中额外说明用途，否则审核会重点关注。
